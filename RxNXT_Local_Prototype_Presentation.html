The Local-First Advantage

RxNXT

We pivoted to a zero-dependency architecture. RxNXT is now capable of running completely on local clinic networks without requiring heavy cloud databases or internet connectivity.

  • No Cloud Lock-in: Fully decoupled from Supabase. Tables and data are implemented locally.
  • Instant Response: Zero network latency. Queries run instantly via in-memory structures and local JSON persistence.
  • Complete Data Privacy: Patient data never leaves the physical clinic building.
  • Lightweight Deployment: Runs purely on Node.js locally, or on edge networks with persistent storage mounts.

Hardened & Self-Contained

RxNXT

The new prototype introduces rigorous application-level security and local database management without external services.

Scrypt Security

Role-based PINs are hashed using advanced scrypt with per-PIN salting. Never stored as plain text.

Local Database

Fully functioning JSON database with serialised mutex writes to completely prevent race conditions.

Rate Limiting

Built-in protection: 5 failed attempts locks out the IP/Role for 15 minutes.

Robust Handlers

Strict request size capping (1MB limit), automatic JSON parse error handling, and injected security headers.

Architecture Summary

RxNXT

Deployable Node.js MVP

  • Frontend: Pure HTML/JS/CSS tailored for raw speed.
  • Backend: Self-contained Node.js server (`serve-rxnxt.mjs`).
  • Database: Flat-file `drugs-db.js` & persistent `rxnxt-db.json`.
  • Security: CSP, X-Frame-Options, HSTS, rate-limiting, and Scrypt hashing.

Deployment

Ready for absolute offline execution on clinic hardware, or 1-click cloud deployment via Docker/Railway.
